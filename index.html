Tema 1: Elementos de Páginas Web

  1. Orígenes de la Web

    1. Web 2.0

      2. Se refiere a la evolución de la web estática a una más interactiva y colaborativa,
      donde los usuarios pueden generar contenido, como en redes sociales, blogs y wikis.

    2. Web 3.0 3

      3. Es la futura evolución de la web, también llamada Web Semántica,
      que utiliza inteligencia artificial y tecnologías descentralizadas como blockchain para mejorar la interacción y personalización.

    3. La W3C y los estándares

      4. El World Wide Web Consortium (W3C)
      es una organización internacional que desarrolla estándares web, garantizando la interoperabilidad entre navegadores y dispositivos.

  2. Lenguajes de etiquetas

    1. SGML

      2. Standard Generalized Markup Language,
      un estándar para definir lenguajes de marcas, usado como base para crear otros lenguajes como HTML.

    2. XML

      3. Extensible Markup Language,
      diseñado para almacenar y transportar datos, permitiendo la creación de documentos con una estructura definida.

    3. HTML

      4. HyperText Markup Language, es el lenguaje base
      para crear páginas web. Define la estructura del contenido.

    4. XHTML

      5. Extensible Hypertext Markup Language, una versión más estricta de HTML, basada en XML, que exige cumplir con reglas más
      rigurosas de escritura.

    5. HTML5

      6. La versión más reciente de HTML, con soporte para nuevas funciones como multimedia (audio, video) sin necesidad de
      complementos y una mejor semántica.

  3. Otros lenguajes relacionados al desarrollo web

    1. CSS/CSS3

      2. Cascading Style Sheets, utilizado para definir el diseño y la presentación de páginas web.
      CSS3 es su versión más reciente, con soporte para nuevas propiedades como animaciones y efectos gráficos.

    2. JavaScript

      3. Lenguaje de programación que se ejecuta en el navegador y permite crear interactividad
      en las páginas web, como animaciones, formularios dinámicos, etc.

    3. Ajax

      4. Asynchronous JavaScript and XML, una técnica que permite actualizar partes de una
      página web sin recargar toda la página.

    4. Tecnologías y lenguajes del lado del servidor

      5. ncluyen lenguajes como PHP, Python, Ruby, y frameworks como Node.js, que procesan la lógica y
      gestionan la base de datos en el servidor antes de enviar la respuesta al cliente.

    5. Navegadores

      6. Programas como Chrome, Firefox, Safari, Edge, que permiten acceder, visualizar e interactuar con páginas web.

    6. Motores renderizados

      7. Componentes en los navegadores que procesan el código HTML, CSS y JavaScript para mostrar las
      páginas web en pantalla. Ejemplos son Blink (Chrome), WebKit (Safari) y Gecko (Firefox).

  4. Herramientas de desarrollo

    1. Software WYSIWYG

      2. "What You See Is What You Get" es un tipo de editor que permite
      crear páginas web visualmente sin escribir código, como Adobe Dreamweaver.

    2. Otras alternativas

      1. Figma

        2. Herramienta basada en la nube para diseño de interfaces de usuario,
        que permite la colaboración en tiempo real.

      2. InVisión

        3. Plataforma de diseño y prototipos, que facilita la creación y validación de ideas
        antes de la implementación.

      3. Zeplin

        4. Herramienta que facilita la colaboración entre diseñadores y
        desarrolladores, proporcionando guías de estilo y especificaciones.

      4. GitHub

        5. Plataforma de control de versiones y colaboración en proyectos de desarrollo de software, que permite gestionar
        el código y trabajar en equipo.

    3. Inspección de código desde el navegador.

      4. Herramienta disponible en la mayoría de los navegadores web (como Chrome, Firefox) que permite a los desarrolladores ver y editar
      temporalmente el código HTML, CSS y JavaScript de una página web en tiempo real.
      Es útil para depurar errores, optimizar el rendimiento y probar diseños.

Tema 2: El Diseño De Una Página Web

  1. Tipos de páginas Web

    • Páginas Estáticas:

      • Su contenido no cambia en función de la interacción del usuario y
      se entregan tal como fueron desarrolladas.

    • Páginas Dinámicas:

      • Generan contenido en tiempo real, dependiendo de la interacción del usuario o datos externos,
      como bases de datos o formularios.

    • Páginas Responsivas:

      • Se adaptan a diferentes tamaños de pantalla y
      dispositivos, garantizando una experiencia de usuario óptima en móviles, tablets y ordenadores.

  2. Puntos básicos en la elaboración de una página Web.

    • Objetivo y público: Definir el propósito de la página y a quién va dirigida.

    • Diseño atractivo y funcional: Crear un diseño visualmente agradable que facilite la navegación.

    • Optimización: Asegurar que la página cargue rápido y funcione bien en distintos dispositivos

    • Contenido de calidad: Proporcionar información clara, relevante y bien estructurada.

    • Accesibilidad: Asegurar que el sitio sea accesible para personas con discapacidades
      (por ejemplo, con lectores de pantalla).

Tema 3: Estructura De Una Página Web

  1. Jerarquía de directorio de un proyecto web.

    • index.html: Página principal del sitio.

    • css/: Carpeta que almacena archivos de hojas de estilo.

    • js/: Carpeta para archivos JavaScript.

    • images/:Carpeta que contiene las imágenes usadas en el sitio. Mantener una jerarquía
      ordenada facilita la gestión del proyecto y su escalabilidad.

  2. Elementos de estructura para el diseño de una página Web: Concepto

    Los elementos estructurales definen la disposición y organización de los
    componentes en una página web, haciendo que sea más fácil de usar y entender.

    1. Regla utilizada para validar y certificar una página web

      2. Consiste en cumplir con los estándares del W3C (World Wide Web Consortium), que permiten
      validar que el código HTML y CSS esté bien escrito y siga las buenas prácticas para asegurar compatibilidad y accesibilidad.

    2. Reglas para escribir HTML.

      • Semántica: Usar etiquetas HTML correctamente, como header, article, section, para dar significado al contenido.

      • Anidamiento correcto: Asegurarse de que las etiquetas HTML estén correctamente anidadas, es decir, que cada etiqueta abierta sea cerrada en el orden correcto.

      • Atributos correctos: Utilizar atributos adecuados para cada etiqueta (por ejemplo, alt en imágenes).

      • Especificación de idioma: Declarar el idioma del documento usando el atributo lang en la etiqueta html.

    3. Estructura básica de un archivo web

      • !DOCTYPE html: Define el tipo de documento (HTML5).

      • html lang="es": La etiqueta raíz que contiene el resto del documento.

      • head: Contiene metadatos como el título, enlaces a hojas de estilo, y scripts.

      • title: Define el título que aparecerá en la pestaña del navegador.

      • body: Contiene el contenido visible de la página (texto, imágenes, enlaces, etc.).
